HerdFinder

Find your squad among thousands at Rasrang'26 — without losing your mind.

Background

Rasrang'26 draws thousands of attendees across multiple event zones — stages, food courts, galleries, and arenas. Losing your group in the crowd is inevitable. Constant phone calls drain batteries, and GPS-sharing apps require accounts nobody wants to create at a festival. There has to be a better way.

Problem Statement

Build HerdFinder — a real-time, mobile-friendly web app that lets Rasrang'26 attendees create a private squad, share their approximate zone location, and instantly broadcast a "Find Me" alert to teammates — all without login, GPS permission, or any external dependency.

Core Requirements
1 Squad Creation & Joining
  • User enters their name and creates a squad — gets a 4–6 character invite code.
  • Other users enter the code to join the same squad. No login or account required.
  • Squad must persist across page refreshes (local storage / URL state acceptable).
2 Zone-Based Location Sharing
  • Provide a predefined list of Rasrang'26 zones: Main Stage, Food Court, Art Gallery, Registration Desk, Parking, Game Zone, Backstage, Exit Gate.
  • User selects their current zone from a dropdown or tappable map grid. No GPS required.
  • Display each squad member's name, zone, and last-updated timestamp.
3 "Find Me" SOS Broadcast
  • One-tap Find Me button sends a highlighted alert visible to all squad members.
  • Alert must visually stand out — full-screen flash, sound, or persistent banner.
  • Alert auto-clears after 60 seconds or when manually dismissed.
4 Festival-Grade UX Constraint
  • Must be fully usable on a mobile browser in bright sunlight: high contrast, large tap targets (min 44px), minimal text input.
  • Core flow (join squad → set zone → broadcast) must complete in under 30 seconds for a new user.
Expected Deliverables
🌐Working web app (any stack — HTML/CSS/JS or framework)
📱Mobile-responsive layout — test on phone before submitting
🔗Hosted link (Vercel / Netlify) OR single runnable .html file
🗂️Brief README: Features built + any limitations
Bonus (Optional)
  • Interactive Rasrang'26 zone mini-map with clickable areas.
  • Emoji reactions squad members can send each other (👋 😂 🔥).
  • Squad activity feed showing a log of zone updates and alerts.
Judging Criteria
40% Functionality
30% UX & Design
20% Creativity
10% Code Quality

RasrangEats

Order food from 20+ stalls. Skip the queue. Get your token. Eat.

Background

Rasrang'26's food zone hosts 20+ stalls serving thousands of attendees over a single day. Long queues form within minutes of stalls opening — often with no menu visibility from a distance, no order tracking, and no way to know which stall still has stock. The chaos is real, and so is the hunger.

Problem Statement

Build RasrangEats — a two-sided web platform where food stall vendors can digitally list their menu and manage incoming orders, and customers can browse all stalls, place pre-orders, and receive a queue token for pickup — all without payment gateways, logins, or backends.

Core Requirements — Vendor Side
V1 Stall Registration
  • Vendor enters: Stall Name, Cuisine Type (e.g., South Indian, Beverages, Snacks), Stall Number / Location.
  • After registration, vendor lands on their Stall Dashboard.
V2 Menu Management
  • Add / edit / remove menu items: Item Name, Price (₹), Availability toggle (In Stock / Sold Out).
  • Sold-out items must be clearly visible to customers but non-orderable.
V3 Live Order Dashboard
  • Vendor sees incoming orders in a FIFO queue with customer name, items ordered, and order timestamp.
  • Vendor can move each order through: Pending → Preparing → Ready for Pickup.
  • Completed orders collapse to a history log.
Core Requirements — Customer Side
C1 Stall Browser
  • Grid/list view of all registered stalls with cuisine type and open/busy status.
  • Filter by cuisine type or "In Stock" availability.
  • Tap a stall → see full menu with prices and stock status.
C2 Cart & Pre-Order
  • Add items from one stall at a time to cart (cross-stall ordering not required).
  • Checkout: enter Name + Phone Number. No login required. Cash-on-pickup model.
  • On submit — generate a unique order token (e.g., #FD042).
C3 Token & Wait Time
  • Post-order screen shows: Order Token, Items Ordered, Stall Name & Location.
  • Display estimated wait time — calculated as (pending orders ahead × 2 minutes).
  • Customer can re-enter token to check live status: Pending / Preparing / Ready ✅.
Expected Deliverables
🌐Working web app (any stack — HTML/CSS/JS or framework)
🔀Both Vendor & Customer views accessible from same app (toggle/switch in UI)
💾Local storage or in-memory state acceptable — no backend required
🔗Hosted link (Vercel / Netlify) OR single runnable .html file
Bonus (Optional)
  • Printable QR code per stall that customers can scan to jump directly to that stall's menu.
  • Sales analytics for vendor: total orders, revenue, best-selling item.
  • Dark / festive UI theme inspired by Rasrang'26 branding.
